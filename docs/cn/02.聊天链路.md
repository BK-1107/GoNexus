# 02.聊天链路

本篇只讲当前前端主路径：SSE 流式聊天。

## 1.完整链路

```text
ChatInput.tsx
  -> useStreaming.ts
  -> POST /api/v1/AI/chat/send-stream 或 /send-stream-new-session
  -> Router
  -> JWT middleware
  -> controller
  -> service
  -> AIHelperManager
  -> AIModelFactory
  -> AIHelper
  -> AIModel.StreamResponse
  -> SSE data: xxx\n\n 返回前端
  -> frontend updateLastMessage
  -> RabbitMQ
  -> dao/message
  -> MySQL
```

对应到项目文件：

```text
GoNexus/frontend/src/components/chat/ChatInput.tsx
  -> GoNexus/frontend/src/hooks/useStreaming.ts
  -> GoNexus/router/AI.go
  -> GoNexus/middleware/jwt/jwt.go
  -> GoNexus/controller/session/session.go
  -> GoNexus/service/session/session.go
  -> GoNexus/common/aihelper/manager.go
  -> GoNexus/common/aihelper/factory.go
  -> GoNexus/common/aihelper/aihelper.go
  -> GoNexus/common/aihelper/model.go
  -> GoNexus/common/rabbitmq/meesage.go
  -> GoNexus/dao/message/message.go
  -> GoNexus/model/message.go
```

```go
func CreateStreamSessionOnly(userName string, userQuestion string) (string, code.Code) {
	newSession := &model.Session{
		// 每个聊天窗口一个唯一 ID
		ID: uuid.New().String(),

		// 绑定当前登录用户
		UserName: userName,

		// 默认用第一条问题当会话标题
		Title: userQuestion,
	}

	// 通过 dao 写入数据库
	createdSession, err := daosession.CreateSession(newSession)
	if err != nil {
		log.Println("CreateStreamSessionOnly CreateSession error:", err)
		return "", code.CodeServerBusy
	}

	// 返回新创建的 sessionId
	return createdSession.ID, code.CodeSuccess
}
```

## 13.AIHelper：保存上下文并调用模型

文件：`GoNexus/common/aihelper/aihelper.go`

```go
func (a *AIHelper) StreamResponse(
	userName string,
	ctx context.Context,
	cb StreamCallback,
	userQuestion string,
) (*model.Message, error) {
	// 先把用户问题加入当前会话上下文，并保存
	a.AddMessage(userQuestion, userName, true, true)

	// 转成大模型 SDK 需要的消息格式
	messages := utils.ConvertToSchemaMessages(a.messages)

	// 调用具体模型的流式输出
	content, err := a.model.StreamResponse(ctx, messages, cb)
	if err != nil {
		return nil, err
	}

	// 组装 AI 回复消息
	modelMsg := &model.Message{
		SessionID: a.SessionID,
		UserName:  userName,
		Content:   content,
		IsUser:    false,
	}

	// 把 AI 回复也加入上下文，并保存
	a.AddMessage(modelMsg.Content, userName, false, true)
	return modelMsg, nil
}
```

## 14.AIModel：流式调用模型

文件：`GoNexus/common/aihelper/model.go`

```go
func (o *OpenAIModel) StreamResponse(
	ctx context.Context,
	messages []*schema.Message,
	cb StreamCallback,
) (string, error) {
	// 调用模型 SDK 的流式接口
	stream, err := o.llm.Stream(ctx, messages)
	if err != nil {
		return "", err
	}
	defer stream.Close()

	content := ""
	for {
		// 不断接收模型返回的片段
		chunk, err := stream.Recv()
		if err != nil {
			break
		}

		// 拼出完整回答
		content += chunk.Content

		// 把当前片段交给 service，service 会写成 SSE 返回前端
		cb(chunk.Content)
	}

	return content, nil
}
```

## 15.RabbitMQ：消息进入队列

文件：`GoNexus/common/aihelper/aihelper.go`

```go
func (a *AIHelper) AddMessage(Content string, UserName string, IsUser bool, Save bool) {
	userMsg := model.Message{
		SessionID: a.SessionID,
		Content:   Content,
		UserName:  UserName,
		IsUser:    IsUser,
	}

	// 先放到当前会话的内存上下文
	a.messages = append(a.messages, &userMsg)

	// Save 为 true 时，进入持久化流程
	if Save {
		a.saveFunc(&userMsg)
	}
}
```

```go
saveFunc: func(msg *model.Message) (*model.Message, error) {
	// 生成 RabbitMQ 消息参数
	data := rabbitmq.GenerateMessageMQParam(msg.SessionID, msg.Content, msg.UserName, msg.IsUser)

	// 发布到 RabbitMQ，后续由消费者写入 MySQL
	err := rabbitmq.RMQMessage.Publish(data)
	return msg, err
}
```

## 17.Frontend：更新最后一条 AI 消息

文件：`GoNexus/frontend/src/store/chatStore.ts`

```ts
updateLastMessage: (content) => set((state) => {
  const newMessages = [...state.messages]
  if (newMessages.length > 0) {
    // 最后一条消息就是前面预先插入的空 AI 消息
    newMessages[newMessages.length - 1] = { 
      ...newMessages[newMessages.length - 1], 
      // 用最新累积内容替换它
      content 
    }
  }
  return { messages: newMessages }
})
```
