# 03.会话与消息持久化

会话与消息持久化负责把用户消息和 AI 回复保存下来。
本篇讲消息保存链路，也就是 AIHelper 如何先保存内存上下文，再通过 RabbitMQ 异步写入 MySQL。

## 1.完整链路

```text
AIHelper.AddMessage
  -> 内存上下文 a.messages
  -> saveFunc
  -> GenerateMessageMQParam
  -> RabbitMQ Publish
  -> MQMessage consumer
  -> dao/message
  -> MySQL
```

对应到项目文件：

```text
GoNexus/common/aihelper/aihelper.go
  -> GoNexus/common/rabbitmq/meesage.go
  -> GoNexus/common/rabbitmq/rabbitmq.go
  -> GoNexus/dao/message/message.go
  -> GoNexus/common/mysql/mysql.go
  -> GoNexus/model/message.go
```

## 2.AIHelper：创建保存回调

文件：`GoNexus/common/aihelper/aihelper.go`

```go
func NewAIHelper(model_ AIModel, SessionID string) *AIHelper {
	return &AIHelper{
		model:    model_,
		messages: make([]*model.Message, 0),

		// 每次 AddMessage 需要保存时，会调用这个函数
		saveFunc: func(msg *model.Message) (*model.Message, error) {
			// 把 message 转成 RabbitMQ 消息体
			data := rabbitmq.GenerateMessageMQParam(msg.SessionID, msg.Content, msg.UserName, msg.IsUser)

			// 发布到 RabbitMQ，真正写库交给消费者异步处理
			err := rabbitmq.RMQMessage.Publish(data)
			return msg, err
		},

		// 一个聊天会话绑定一个 AIHelper
		SessionID: SessionID,
	}
}
```

## 3.AIHelper：消息进入内存上下文

文件：`GoNexus/common/aihelper/aihelper.go`

```go
func (a *AIHelper) AddMessage(Content string, UserName string, IsUser bool, Save bool) {
	userMsg := model.Message{
		// 当前消息属于哪个聊天会话
		SessionID: a.SessionID,

		// 消息正文
		Content: Content,

		// 当前用户
		UserName: UserName,

		// true 是用户消息，false 是 AI 消息
		IsUser: IsUser,
	}

	// 先放进当前 AIHelper 的内存上下文
	a.messages = append(a.messages, &userMsg)

	// Save 为 true 时，才进入持久化流程
	if Save {
		a.saveFunc(&userMsg)
	}
}
```

## 4.AIHelper：用户消息和 AI 回复都会保存

文件：`GoNexus/common/aihelper/aihelper.go`

```go
func (a *AIHelper) StreamResponse(userName string, ctx context.Context, cb StreamCallback, userQuestion string) (*model.Message, error) {
	// 保存用户问题
	a.AddMessage(userQuestion, userName, true, true)

	a.mu.RLock()
	messages := utils.ConvertToSchemaMessages(a.messages)
	a.mu.RUnlock()

	// 调用模型生成 AI 回复
	content, err := a.model.StreamResponse(ctx, messages, cb)
	if err != nil {
		return nil, err
	}

	modelMsg := &model.Message{
		SessionID: a.SessionID,
		UserName:  userName,
		Content:   content,
		IsUser:    false,
	}

	// 保存 AI 回复
	a.AddMessage(modelMsg.Content, userName, false, true)

	return modelMsg, nil
}
```

## 5.RabbitMQ：生成消息体

文件：`GoNexus/common/rabbitmq/meesage.go`

```go
type MessageMQParam struct {
	// 会话 ID
	SessionID string `json:"session_id"`

	// 消息正文
	Content string `json:"content"`

	// 用户名
	UserName string `json:"user_name"`

	// 是否为用户消息
	IsUser bool `json:"is_user"`
}
```

```go
func GenerateMessageMQParam(sessionID string, content string, userName string, IsUser bool) []byte {
	param := MessageMQParam{
		SessionID: sessionID,
		Content:   content,
		UserName:  userName,
		IsUser:    IsUser,
	}

	// 转成 JSON 字节，作为 RabbitMQ 消息体
	data, _ := json.Marshal(param)
	return data
}
```

## 6.RabbitMQ：消费消息并写库

文件：`GoNexus/common/rabbitmq/meesage.go`

```go
func MQMessage(msg *amqp.Delivery) error {
	var param MessageMQParam

	// 从 RabbitMQ 消息体解析出消息参数
	err := json.Unmarshal(msg.Body, &param)
	if err != nil {
		return err
	}

	// 转成数据库 message model
	newMsg := &model.Message{
		SessionID: param.SessionID,
		Content:   param.Content,
		UserName:  param.UserName,
		IsUser:    param.IsUser,
	}

	// 消费者异步插入到数据库中
	message.CreateMessage(newMsg)
	return nil
}
```

## 8.Model：消息表结构

文件：`GoNexus/model/message.go`

```go
type Message struct {
	// 消息主键
	ID uint `gorm:"primaryKey;autoIncrement" json:"id"`

	// 这条消息属于哪个聊天会话
	SessionID string `gorm:"index;not null;type:varchar(36)" json:"session_id"`

	// 这条消息属于哪个用户
	UserName string `gorm:"type:varchar(20)" json:"username"`

	// 消息正文
	Content string `gorm:"type:text" json:"content"`

	// true 表示用户消息，false 表示 AI 消息
	IsUser bool `gorm:"not null;" json:"is_user"`

	CreatedAt time.Time `json:"created_at"`
}
```
