# 06. Docker デプロイ連携

Docker デプロイは、フロントエンド、バックエンド、MySQL、Redis Stack、RabbitMQ をまとめて起動します。
この章では、`docker-compose.yml` が GoNexus 全体をどのように起動し、コンテナ同士がどのように通信するかを説明します。

## 1. 全体の流れ

```text
docker-compose up -d
  -> frontend コンテナ
  -> backend コンテナ
  -> mysql コンテナ
  -> redis-stack コンテナ
  -> rabbitmq コンテナ
  -> backend が config.toml と環境変数を読む
  -> backend が MySQL / Redis / RabbitMQ に接続する
  -> frontend Nginx が /api を backend:9090 へ proxy する
```

対応するファイル：

```text
GoNexus/docker-compose.yml
  -> GoNexus/Dockerfile
  -> GoNexus/frontend/Dockerfile
  -> GoNexus/frontend/nginx.conf
  -> GoNexus/config/config.go
  -> GoNexus/main.go
  -> GoNexus/common/mysql/mysql.go
  -> GoNexus/common/redis/redis.go
  -> GoNexus/common/rabbitmq/rabbitmq.go
```

## 2. Docker Compose：全サービスを起動する

ファイル：`GoNexus/docker-compose.yml`

```yaml
services:
  frontend:
    # フロントエンド image を build する
    build:
      context: ./frontend
      dockerfile: Dockerfile
    container_name: gonexus_frontend
    restart: always
    ports:
      # host 8080 -> container 80
      - "8080:80"
    depends_on:
      # frontend は backend に依存する
      - backend
```

```yaml
  backend:
    # Go backend image を build する
    build:
      context: .
      dockerfile: Dockerfile
    container_name: gonexus_backend
    restart: always
    ports:
      # host 9090 -> container 9090
      - "9090:9090"
```

## 3. Backend：環境変数

ファイル：`GoNexus/docker-compose.yml`

```yaml
environment:
  # Gin の listen address
  GONEXUS_HOST: 0.0.0.0
  GONEXUS_PORT: 9090

  # backend は Docker network 内で mysql:3306 に接続する
  GONEXUS_MYSQL_HOST: mysql
  GONEXUS_MYSQL_PORT: 3306
  GONEXUS_MYSQL_USER: root
  GONEXUS_MYSQL_PASSWORD: root
  GONEXUS_MYSQL_DATABASE: GoNexus

  # Redis Stack service name
  GONEXUS_REDIS_HOST: redis
  GONEXUS_REDIS_PORT: 6379

  # RabbitMQ service name
  GONEXUS_RABBITMQ_HOST: rabbitmq
  GONEXUS_RABBITMQ_PORT: 5672
  GONEXUS_RABBITMQ_USERNAME: root
  GONEXUS_RABBITMQ_PASSWORD: 123456
```

## 4. Backend：volume mount

ファイル：`GoNexus/docker-compose.yml`

```yaml
volumes:
  # RAG のアップロードファイルを host の GoNexus/uploads に保存する
  - ./uploads:/app/uploads

  # backend log を host の GoNexus/logs に保存する
  - ./logs:/app/logs

  # local config.toml を read-only で container に mount する
  - ./config/config.toml:/app/config/config.toml:ro
```

## 5. ミドルウェアコンテナ

ファイル：`GoNexus/docker-compose.yml`

```yaml
mysql:
  image: mysql:8.0
  environment:
    # root password
    MYSQL_ROOT_PASSWORD: root

    # default database
    MYSQL_DATABASE: GoNexus
  volumes:
    # MySQL data を永続化する
    - mysql_data:/var/lib/mysql
```

```yaml
redis:
  image: redis/redis-stack:latest
  ports:
    # Redis port
    - "6379:6379"

    # Redis Stack UI
    - "8001:8001"
  volumes:
    # Redis data を永続化する
    - redis_data:/data
```

```yaml
rabbitmq:
  image: rabbitmq:3-management
  environment:
    # RabbitMQ login
    RABBITMQ_DEFAULT_USER: root
    RABBITMQ_DEFAULT_PASS: 123456
  ports:
    # AMQP port
    - "5672:5672"

    # management UI
    - "15672:15672"
  volumes:
    # RabbitMQ data を永続化する
    - rabbitmq_data:/var/lib/rabbitmq
```

## 6. Backend Dockerfile：Go サービスを build する

ファイル：`GoNexus/Dockerfile`

```dockerfile
FROM golang:1.24-bookworm AS builder

WORKDIR /src

# go.mod と go.sum を先にコピーして dependency download を cache する
COPY go.mod go.sum ./
RUN go mod download

# backend source code をコピーする
COPY . .

# Linux binary を build する
RUN CGO_ENABLED=0 GOOS=linux go build -tags=nomsgpack -trimpath -ldflags="-s -w -checklinkname=0" -o /out/gonexus .
```

```dockerfile
FROM gcr.io/distroless/static-debian12:nonroot

WORKDIR /app

# runtime image には build 済み binary だけをコピーする
COPY --from=builder /out/gonexus /app/gonexus

EXPOSE 9090

# container 起動時に backend service を実行する
ENTRYPOINT ["/app/gonexus"]
```

## 7. Frontend Dockerfile と Nginx proxy

ファイル：`GoNexus/frontend/Dockerfile`、`GoNexus/frontend/nginx.conf`

```dockerfile
FROM node:24-alpine AS builder

WORKDIR /app

# frontend dependencies を install する
COPY package*.json ./
RUN npm ci

# dist 静的ファイルを build する
COPY . .
RUN npm run build
```

```dockerfile
FROM nginx:1.29-alpine

# custom Nginx config を使う
COPY nginx.conf /etc/nginx/conf.d/default.conf

# Vite output を Nginx static directory へコピーする
COPY --from=builder /app/dist /usr/share/nginx/html
```

```nginx
location /api/ {
    # Compose 内部 network では service name backend でアクセスする
    proxy_pass http://backend:9090/api/;

    # SSE のため buffering を無効化する
    proxy_buffering off;

    # streaming chat は長時間続く場合がある
    proxy_read_timeout 3600s;
}
```

## 8. Config と backend startup

ファイル：`GoNexus/config/config.go`、`GoNexus/main.go`

```go
func InitConfig() error {
	// default config path
	configPath := os.Getenv("GONEXUS_CONFIG_PATH")
	if configPath == "" {
		configPath = "config/config.toml"
	}

	// config.toml があれば先に読み込む
	if _, err := os.Stat(configPath); err == nil {
		if _, err := toml.DecodeFile(configPath, config); err != nil {
			return err
		}
	}

	// その後、環境変数で上書きする
	applyEnvOverrides(config)
	return nil
}
```

```go
func main() {
	// log を初期化する
	logFile, err := initLogging()
	if err != nil {
		return
	}
	defer logFile.Close()

	// config を読む
	conf := config.GetConfig()

	// MySQL と AutoMigrate を初期化する
	mysql.InitMysql()

	// Redis を初期化する
	redis.Init()

	// RabbitMQ を初期化し、consumer を起動する
	rabbitmq.InitRabbitMQ()

	// Gin HTTP server を起動する
	StartServer(conf.MainConfig.Host, conf.MainConfig.Port)
}
```

## 9. コマンド

```bash
# backend project directory へ移動する
cd GoNexus

# すべての container を build して background 起動する
docker-compose up -d

# container 状態を確認する
docker-compose ps

# backend log を追跡する
docker-compose logs -f backend

# すべての container を停止する
docker-compose down
```
