# 06. Docker部署链路

Docker 部署负责把前端、后端、MySQL、Redis Stack、RabbitMQ 一次性拉起。
本篇讲 `docker-compose.yml` 如何启动整套 GoNexus 服务，以及容器之间如何互相访问。

## 1.完整链路

```text
docker-compose up -d
  -> frontend 容器
  -> backend 容器
  -> mysql 容器
  -> redis-stack 容器
  -> rabbitmq 容器
  -> backend 读取 config.toml 和环境变量
  -> backend 连接 MySQL / Redis / RabbitMQ
  -> frontend Nginx 代理 /api 到 backend:9090
```

对应到项目文件：

```text
GoNexus/docker-compose.yml
  -> GoNexus/Dockerfile
  -> GoNexus/frontend/Dockerfile
  -> GoNexus/frontend/nginx.conf
  -> GoNexus/config/config.go
  -> GoNexus/main.go
  -> GoNexus/common/mysql/mysql.go
  -> GoNexus/common/redis/redis.go
  -> GoNexus/common/rabbitmq/rabbitmq.go
```

## 2.Docker Compose：一次启动所有服务

文件：`GoNexus/docker-compose.yml`

```yaml
services:
  frontend:
    # 构建前端镜像
    build:
      context: ./frontend
      dockerfile: Dockerfile
    container_name: gonexus_frontend
    restart: always
    ports:
      # 宿主机 8080 -> 容器 80
      - "8080:80"
    depends_on:
      # 前端依赖后端先启动
      - backend
```

```yaml
  backend:
    # 构建 Go 后端镜像
    build:
      context: .
      dockerfile: Dockerfile
    container_name: gonexus_backend
    restart: always
    ports:
      # 宿主机 9090 -> 容器 9090
      - "9090:9090"
```

## 3.Backend：容器环境变量

文件：`GoNexus/docker-compose.yml`

```yaml
environment:
  # Gin 服务监听地址
  GONEXUS_HOST: 0.0.0.0
  GONEXUS_PORT: 9090

  # MySQL 容器服务名，backend 通过 mysql:3306 访问
  GONEXUS_MYSQL_HOST: mysql
  GONEXUS_MYSQL_PORT: 3306
  GONEXUS_MYSQL_USER: root
  GONEXUS_MYSQL_PASSWORD: root
  GONEXUS_MYSQL_DATABASE: GoNexus

  # Redis Stack 容器服务名
  GONEXUS_REDIS_HOST: redis
  GONEXUS_REDIS_PORT: 6379

  # RabbitMQ 容器服务名
  GONEXUS_RABBITMQ_HOST: rabbitmq
  GONEXUS_RABBITMQ_PORT: 5672
  GONEXUS_RABBITMQ_USERNAME: root
  GONEXUS_RABBITMQ_PASSWORD: 123456
```

## 4.Backend：挂载目录

文件：`GoNexus/docker-compose.yml`

```yaml
volumes:
  # RAG 上传文件保存到宿主机 GoNexus/uploads
  - ./uploads:/app/uploads

  # 后端日志保存到宿主机 GoNexus/logs
  - ./logs:/app/logs

  # 本地 config.toml 只读挂载进容器
  - ./config/config.toml:/app/config/config.toml:ro
```

## 6.MySQL：数据落库容器

文件：`GoNexus/docker-compose.yml`

```yaml
mysql:
  image: mysql:8.0
  container_name: gopherai_mysql
  restart: always
  environment:
    # root 密码
    MYSQL_ROOT_PASSWORD: root

    # 默认创建 GoNexus 数据库
    MYSQL_DATABASE: GoNexus
  ports:
    - "3306:3306"
  volumes:
    # MySQL 数据持久化到 Docker volume
    - mysql_data:/var/lib/mysql
```

## 7.Redis Stack：RAG 向量索引容器

文件：`GoNexus/docker-compose.yml`

```yaml
redis:
  image: redis/redis-stack:latest
  container_name: gopherai_redis
  restart: always
  ports:
    # Redis 访问端口
    - "6379:6379"

    # Redis Stack 管理页面
    - "8001:8001"
  volumes:
    # Redis 数据持久化
    - redis_data:/data
```

## 8.RabbitMQ：消息队列容器

文件：`GoNexus/docker-compose.yml`

```yaml
rabbitmq:
  image: rabbitmq:3-management
  container_name: gopherai_rabbitmq
  restart: always
  environment:
    # RabbitMQ 登录账号密码
    RABBITMQ_DEFAULT_USER: root
    RABBITMQ_DEFAULT_PASS: 123456
  ports:
    # AMQP 连接端口
    - "5672:5672"

    # RabbitMQ 管理后台
    - "15672:15672"
  volumes:
    # RabbitMQ 数据持久化
    - rabbitmq_data:/var/lib/rabbitmq
```

## 9.Backend Dockerfile：构建 Go 服务

文件：`GoNexus/Dockerfile`

```dockerfile
FROM golang:1.24-bookworm AS builder

WORKDIR /src

# 先复制 go.mod 和 go.sum，利用 Docker 缓存下载依赖
COPY go.mod go.sum ./
RUN go mod download

# 复制后端源码
COPY . .

# 编译 Linux 可执行文件
RUN CGO_ENABLED=0 GOOS=linux go build -tags=nomsgpack -trimpath -ldflags="-s -w -checklinkname=0" -o /out/gonexus .
```

```dockerfile
FROM gcr.io/distroless/static-debian12:nonroot

WORKDIR /app

# 只把编译后的二进制复制进运行镜像
COPY --from=builder /out/gonexus /app/gonexus

EXPOSE 9090

# 容器启动时运行后端服务
ENTRYPOINT ["/app/gonexus"]
```

## 10.Frontend Dockerfile：构建静态页面

文件：`GoNexus/frontend/Dockerfile`

```dockerfile
FROM node:24-alpine AS builder

WORKDIR /app

# 安装前端依赖
COPY package*.json ./
RUN npm ci

# 构建 dist 静态文件
COPY . .
RUN npm run build
```

```dockerfile
FROM nginx:1.29-alpine

# 使用自定义 Nginx 配置
COPY nginx.conf /etc/nginx/conf.d/default.conf

# 把 Vite 构建结果放到 Nginx 静态目录
COPY --from=builder /app/dist /usr/share/nginx/html

EXPOSE 80
```

## 11.Nginx：前端代理后端 API

文件：`GoNexus/frontend/nginx.conf`

```nginx
location /api/ {
    # Docker Compose 内部通过服务名 backend 访问后端
    proxy_pass http://backend:9090/api/;

    # SSE 流式响应需要关闭缓冲
    proxy_buffering off;

    # 流式聊天可能持续较久，读超时设置长一点
    proxy_read_timeout 3600s;
}
```

```nginx
location / {
    # React 单页应用刷新时回退到 index.html
    try_files $uri $uri/ /index.html;
}
```

## 12.Config：读取 config.toml 和环境变量

文件：`GoNexus/config/config.go`

```go
func InitConfig() error {
	// 默认读取 config/config.toml
	configPath := os.Getenv("GONEXUS_CONFIG_PATH")
	if configPath == "" {
		configPath = "config/config.toml"
	}

	// 如果存在 config.toml，就先加载本地配置
	if _, err := os.Stat(configPath); err == nil {
		if _, err := toml.DecodeFile(configPath, config); err != nil {
			return err
		}
	}

	// 再用环境变量覆盖配置
	applyEnvOverrides(config)
	return nil
}
```

## 13.Main：后端启动顺序

文件：`GoNexus/main.go`

```go
func main() {
	// 初始化日志，写入控制台和 logs/backend.log
	logFile, err := initLogging()
	if err != nil {
		return
	}
	defer logFile.Close()

	// 读取配置
	conf := config.GetConfig()
	host := conf.MainConfig.Host
	port := conf.MainConfig.Port

	// 初始化 MySQL，并执行 AutoMigrate
	if err := mysql.InitMysql(); err != nil {
		return
	}

	// 初始化 Redis
	redis.Init()

	// 初始化 RabbitMQ，并启动消息消费者
	rabbitmq.InitRabbitMQ()

	// 启动 Gin HTTP 服务
	StartServer(host, port)
}
```

## 14.启动命令

```bash
# 进入后端项目目录
cd GoNexus

# 构建并后台启动所有容器
docker-compose up -d

# 查看容器状态
docker-compose ps

# 查看后端日志
docker-compose logs -f backend

# 停止所有容器
docker-compose down
```
